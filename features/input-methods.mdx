## Telex Input Method

### Overview

Telex uses **letter keys** that don't exist in Vietnamese (`f`, `j`, `w`, `z`) to add diacritics.

<CodeGroup>

```text Tone Marks
s = sắc   (acute)
f = huyền (grave)
r = hỏi   (hook)
x = ngã   (tilde)
j = nặng  (dot below)
z = remove last diacritic
```

```text Modifiers
aa = â  (circumflex on a)
ee = ê  (circumflex on e)
oo = ô  (circumflex on o)
aw = ă  (breve on a)
ow = ơ  (horn on o)
uw = ư  (horn on u)
dd = đ  (stroke on d)
```

</CodeGroup>

### Undo Mechanism

Type the same key twice to undo:

```
Input    Output    Explanation
───────────────────────────────
aaa      âa        aa → â, third 'a' stays
ass      as        as → á, second 's' removes sắc
aww      aw        aw → ă, second 'w' removes breve
oww      ow        ow → ơ, second 'w' removes horn
```

<Note>
**Issue #312**: When a vowel already has a modifier (horn/circumflex/breve), typing the same vowel again adds it **raw**, not as a modifier.

Example: `chưa` + `a` → `chưaa` (NOT `chưâ`)
</Note>

### Implementation

Telex is implemented in `core/src/input/telex.rs`:

```rust
pub struct Telex;

impl Method for Telex {
    fn mark(&self, key: u16) -> Option<u8> {
        match key {
            keys::S => Some(1), // sắc
            keys::F => Some(2), // huyền
            keys::R => Some(3), // hỏi
            keys::X => Some(4), // ngã
            keys::J => Some(5), // nặng
            _ => None,
        }
    }

    fn tone(&self, key: u16) -> Option<ToneType> {
        match key {
            keys::A | keys::E | keys::O => Some(ToneType::Circumflex),
            keys::W => Some(ToneType::Horn),
            _ => None,
        }
    }

    fn stroke(&self, key: u16) -> bool {
        key == keys::D
    }

    fn remove(&self, key: u16) -> bool {
        key == keys::Z
    }
}
```

## VNI Input Method

### Overview

VNI uses **number keys** (0-9) to add diacritics.

<CodeGroup>

```text Tone Marks
1 = sắc   (acute)
2 = huyền (grave)
3 = hỏi   (hook)
4 = ngã   (tilde)
5 = nặng  (dot below)
0 = remove last diacritic
```

```text Modifiers
6 = ^  (circumflex: a→â, e→ê, o→ô)
7 = ơ  (horn: o→ơ, u→ư)
8 = ˘  (breve: a→ă)
9 = đ  (stroke: d→đ)
```

</CodeGroup>

### Implementation

VNI is implemented in `core/src/input/vni.rs`:

```rust
pub struct Vni;

impl Method for Vni {
    fn mark(&self, key: u16) -> Option<u8> {
        match key {
            keys::N1 => Some(1), // sắc
            keys::N2 => Some(2), // huyền
            keys::N3 => Some(3), // hỏi
            keys::N4 => Some(4), // ngã
            keys::N5 => Some(5), // nặng
            _ => None,
        }
    }

    fn tone(&self, key: u16) -> Option<ToneType> {
        match key {
            keys::N6 => Some(ToneType::Circumflex),
            keys::N7 => Some(ToneType::Horn),
            keys::N8 => Some(ToneType::Breve),
            _ => None,
        }
    }

    fn stroke(&self, key: u16) -> bool {
        key == keys::N9
    }

    fn remove(&self, key: u16) -> bool {
        key == keys::N0
    }
}
```

## Tone Placement Rules

Gõ Nhanh follows **official Vietnamese tone placement rules** (Quy tắc đặt dấu thanh).

### Rule 1: Single Vowel

Tone mark goes directly on the vowel:

```
bá, bà, bả, bã, bạ
```

### Rule 2: Two Vowels

Check the pattern in this table:

| Pattern | Tone Position | Example          |
|---------|---------------|------------------|
| ai, ay  | **First**     | mái, máy         |
| ao, au  | **First**     | sáo, sáu         |
| âu, ây  | **First**     | cấu, cây         |
| eo, êu  | **First**     | kéo, kêu         |
| ia, iê  | **Second**    | kía, tiếng       |
| iu      | **First**     | tíu              |
| oa, oă  | **Second**    | hoá, xoắn        |
| oe      | **Second**    | khoé             |
| oi, ôi  | **First**     | tói, tôi         |
| ơi      | **First**     | cơi              |
| ua      | **First/Second** | múa / quá     |
| uê, uy  | **Second**    | tuế, thuý        |
| ui, uô  | **First**     | túi, cuốn        |
| ưa, ươ  | **First**     | mứa, được        |
| ưi, ưu  | **First**     | gửi, lưu         |
| yê      | **Second**    | yếu              |

<Note>
**Special case for "ua"**:
- After `q`: tone on **a** (quá, quả)
- Without `q` (open syllable): tone on **u** (múa, cúa)
- With final consonant: tone on **a** (muán, quán)
</Note>

### Rule 3: Three Vowels

Tone mark goes on the **middle vowel** (with modifier if present):

| Pattern | Tone Position | Example       |
|---------|---------------|---------------|
| iêu     | **ê**         | tiêu, kiều    |
| yêu     | **ê**         | yêu           |
| oai     | **a**         | ngoài, loại   |
| oay     | **a**         | xoáy          |
| oeo     | **e**         | khoẻo         |
| uây     | **â**         | khuấy         |
| uôi     | **ô**         | cuối, tuổi    |
| ươi     | **ơ**         | mười, tươi    |
| ươu     | **ơ**         | rượu, hươu    |
| uyê     | **ê**         | quyền, chuyện |

### Old vs New Style

There are two schools of tone placement for some patterns:

| Pattern | Old Style | New Style | Note                    |
|---------|-----------|-----------|-------------------------|
| oa      | hòa       | hoà       | Old: tone on o          |
| oe      | hòe       | hoè       | New: tone on a/e        |
| uy      | thùy      | thuỳ      | (per Bộ GD&ĐT 2018)    |

Gõ Nhanh uses the **new style** by default (following Ministry of Education Decision 1989/QĐ-BGDĐT, 2018).

## Auto-Capitalization

Gõ Nhanh automatically capitalizes the first letter after:

- Period (`.`)
- Exclamation mark (`!`)
- Question mark (`?`)
- Enter key

```
chào bạn. bạn khỏe không?
         ↑ Auto-capitalized
         
Result: Chào bạn. Bạn khỏe không?
```

## FAQ

<AccordionGroup>

<Accordion title="Can I switch between Telex and VNI?">
Yes! Go to **Settings → General → Input Method** and select your preferred method. The change takes effect immediately.
</Accordion>

<Accordion title="Why does 'text' become 'têt'?">
In Telex, `e` + `e` → `ê` and `x` → ngã. Use [Auto-Restore](/features/auto-restore) by pressing Space, or press ESC to restore manually.
</Accordion>

<Accordion title="Which method is more popular?">
Telex is more popular among younger users and programmers. VNI is preferred by those who learned it first or need to avoid conflicts with code editing.
</Accordion>

<Accordion title="Can I customize the key mappings?">
No, key mappings are fixed to follow standard Telex and VNI conventions. This ensures compatibility with other Vietnamese IMEs.
</Accordion>

</AccordionGroup>
